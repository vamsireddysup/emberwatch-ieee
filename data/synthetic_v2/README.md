# Dataset v2: physics-grounded synthetic device thermal data

These files are generated by `src/synthesize_thermal.py` (seeded, fully reproducible). They exist because no public dataset pairs real equipment temperature with real ambient weather and labeled faults. Instead of inventing numbers, v2 simulates a transformer-like device with a standard first-order RC thermal model, driven by real ambient data from five Oregon and Northern California weather stations, with every tunable grounded in measurements from the real ETT transformer dataset (see `src/calibrate_ett.py`).

## Files

One file per station, 2020 through 2023 at 5 minute resolution:

- `device_thermal_hood_river.csv` (ASOS, Columbia Gorge)
- `device_thermal_hillsboro.csv` (ASOS, Portland metro)
- `device_thermal_troutdale.csv` (ASOS, Portland metro)
- `device_thermal_klamath.csv` (RAWS, wildfire-prone Klamath basin, real measured solar)
- `device_thermal_gerber.csv` (RAWS, wildfire-prone south Oregon, real measured solar)

Plus `events_catalog.csv`: one row per injected fault event across all stations.

## Columns (station files)

| column | meaning |
|---|---|
| timestamp | 5-min UTC |
| device_temp_c | sensor-observed simulated device temperature (noise + 0.1 C quantization + rare stuck/dropout artifacts) |
| ambient_temp_c | real station ambient temperature |
| load_factor | ground-truth electrical load 0-1. Debug/aux only. NOT a model input; the real node has only two thermistors |
| fault_type | none, loose_connection, overload, cooling_degradation, thermal_runaway |
| event_id | 0 when no event; joins events_catalog.csv (unique within station) |
| label | Normal, Warming, Anomaly |
| split | train (2020-21), val (2022), test (2023). Chronological; calibrate/train on train only |

## How labels are made

Each segment is simulated twice with identical inputs: a healthy twin and a faulted twin. The difference (excess heating) is ground truth. Anomaly latches when excess reaches 6 C (or rises at 0.3 C/min from at least 2 C) and releases below 2 C, alarm-style. Warming is excess of at least 2 C and not falling. So every event has a physically real Normal to Warming to Anomaly lead-up whose length depends on the fault type, instead of fixed-width label windows.

## Fault types

- loose_connection: contact heating ramps to 8-25 C over 0.5-3 days, then repair
- overload: load times 1.5-2.5 for 2-12 hours
- cooling_degradation: cooling effectiveness degrades over 1.5-4 days, then repair
- thermal_runaway: slow precursor then accelerating exponential burst, capped +60 C, then trip. The fire-risk case

Scheduling is Poisson with a mean gap of 15 days per station; roughly 70-90 events per station over 4 years, 3-7% of rows non-Normal.

## Realism notes (honest limitations)

- Noise and autocorrelation targets come from the real ETT transformer oil temperature series; synthetic noise lands within ~0.05 C of the target.
- Diurnal amplitude is intentionally larger than ETT (about 13-20 C daily vs 5.7 C): a small outdoor pole-mounted device in these climates swings far more than a large indoor oil-filled transformer. The amplitude is inherited from the real ambient data, not invented.
- The 6h autocorrelation is lower than ETT for the same reason (stronger diurnal cycle relative to seasonal drift).
- Ambient gaps longer than 1 hour split the simulation into independent segments; nothing is bridged or interpolated across gaps.

## Reproducing

```
./venv/bin/python src/calibrate_ett.py
./venv/bin/python src/synthesize_thermal.py --station all --seed 42
```

Same seed, same output. See the docstring in `src/synthesize_thermal.py` for the physics constants and their justification.
